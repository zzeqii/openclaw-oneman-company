# 贡献指南

感谢你有兴趣为 OpenClaw One-Man Company OS 做贡献！

## 开发环境设置
1. Fork 并克隆本仓库
2. 安装依赖：`npm install`
3. 确保你安装了最新版本的 OpenClaw：`npm install -g openclaw`

## 贡献类型
我们欢迎各种类型的贡献：

### 🐛 提交 Bug 报告
- 确保该 Bug 尚未被报告
- 详细描述复现步骤
- 提供系统环境信息（OS、Node.js版本、OpenClaw版本）

### ✨ 提交新功能
- 先开启一个 Issue 讨论你想要添加的功能
- 确保代码遵循现有的代码风格
- 为新功能添加必要的文档
- 确保所有测试通过

### 📚 改进文档
- 修复拼写错误
- 补充缺失的文档
- 优化示例代码

### 🧩 贡献新技能
- 技能需要符合 [OpenClaw 技能规范](https://docs.openclaw.ai/skills/spec)
- 技能需要有清晰的 README 文档
- 技能需要包含测试用例

## 代码提交规范
- 提交信息遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范
- 每个提交应该是一个独立的逻辑单元
- 提交前确保代码可以正常运行

## 代码风格
- JavaScript 代码遵循 Standard 规范
- Markdown 文档遵循 Markdownlint 规范
- 保持代码简洁、易读

## 提交 Pull Request
1. 从 `main` 分支创建你的功能分支
2. 提交你的修改
3. 确保 PR 描述清晰地说明了修改的内容和目的
4. 链接到相关的 Issue（如果有的话）
5. 等待代码审查

## 行为准则
- 尊重他人，友好交流
- 接纳不同的观点和意见
- 专注于改进项目，而不是人身攻击

有任何问题，欢迎开启 Issue 讨论！
