# security-center-scan
> **全自动敏感信息扫描与泄漏防护skill**，内置关键词扫描+open-source依赖检查，保证代码提交无敏感信息泄漏，是one-man-company安全合规核心能力

## 功能
- **提交前自动扫描**：推送分支前自动扫描所有代码文件，检查是否有硬编码敏感信息
- **关键词识别**：内置敏感关键词规则：`APP_SECRET`/`app_secret`/`API_KEY`/`api_key`/`ACCESS_TOKEN`/`access_token`/`SECRET_KEY`/`secret_key`/`password`/`pwd`/`AK`/`SK`，识别长度超过20位的硬编码密钥
- **自动open-source依赖扫描**：集成开源依赖扫描，检查依赖是否有已知安全漏洞
- **泄漏修复流程**：发现泄漏自动停止推送，提示修复，指导移除硬编码改用环境变量
- **自动记录安全日志**：每次扫描结果记录到安全中心，方便追溯
- **配合Git flow**：只有扫描通过才能推送分支，从源头阻断泄漏

## Usage
自动触发，每次推送feature分支前自动执行：
1. 扫描所有代码文件，排除`node_modules/`/`dist/`/`.git/`/`*.env.example`
2. 发现硬编码敏感信息 → 停止推送，提示修复，指导改用环境变量
3. 扫描通过 → 正常推送，创建PR
4. 扫描结果记录到安全中心日志

## 规则配置
| 关键词 | 触发条件 |
|----------|----------|
| `app_secret`/`APP_SECRET` | 匹配变量名+硬编码值 |
| `api_key`/`API_KEY` | 匹配变量名+硬编码值 |
| `access_token`/`ACCESS_TOKEN` | 匹配变量名+硬编码值 |
| `secret_key`/`SECRET_KEY` | 匹配变量名+硬编码值 |
| `password`/`pwd` | 匹配变量名+硬编码值 |
| `AK`/`SK` | 匹配缩写+硬编码值 |
- 触发阈值：匹配到变量名且值长度≥20位，判定为敏感信息泄漏

## 集成主动运行引擎
- 主动运行引擎调用 → 创建分支 → 安全扫描 → 扫描通过 → 推送分支 → 创建PR
- 扫描不通过 → 停止推送 → 汇报问题 → 等待修复

## 收益
- 从源头阻断敏感信息泄漏，保证GitHub仓库干净安全

- 作为独立skill可以被所有项目复用，提升GitHub项目安全合规形象，增加核心竞争力
