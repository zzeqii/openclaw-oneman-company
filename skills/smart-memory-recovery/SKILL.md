# smart-memory-recovery
> **全自动记忆恢复与校验技能**，保证session reset后100%准确恢复项目进度，彻底解决AI会话遗忘问题，是one-man-company核心竞争力之一

## 功能
- 三重校验记忆恢复：`项目文档原文` > `Git实际提交` > `旧记忆`，永远以最新文档为准
- 项目类型特殊规则：针对不同项目类型增加专项校验规则，保证进度剧情完全对齐
  - 小说创作：写章节前强制重读最新大纲，确认章节顺序和剧情要点再开始写
  - 代码开发：启动前必须核对实际代码行数、Git提交记录，不依赖旧进度报告
  - 商业化项目：启动前核对最新需求文档，确保方向完全正确
- 错误自动修复：发现记忆错误立即触发全链路重校验，更新长时记忆，记录错误避免再犯
- 多重备份保障：长时记忆 + 当日日志 + Git + 向量数据库四重备份，永不丢失进度

## Usage
自动触发，session重启后自动执行：
1. **第一步：强制重读项目最新大纲/需求文档**，必须完整阅读确认每个章节点位
2. 核对Git实际提交状态
3. 逐章核对已完成内容，确保剧情点位、人物设定完全对齐大纲
4. 对比记忆，发现偏差立即修正
5. 更新长时记忆，记录错误经验

## 小说写作特殊规则
- **写每一章前必须**：
  1. 重读最终定版大纲，找到本章对应的剧情点位
  2. 核对人物设定，确保人设不OOC
  3. 确认本章开头结尾钩子，符合晋江节奏要求
  4. 开始写作
- **session重启后必须**：
  1. 重读大纲全文
  2. 逐章核对已写章节，确保点位完全对齐
  3. 发现错位立即修正重写
  4. 确认正确后继续写下一章

## 核心校验规则（通用）
1. **session重启后必做**：第一步核对最新项目文档，再核对Git，最后看旧记忆
2. **文档优先**：任何时候文档代码比记忆可信，不沿用过期记忆
告诉你 3. **错误闭环**：发现错误立即纠正，更新规则，记录到安全日志
4. **项目专项**：不同项目类型增加专项校验，保证完全对齐
