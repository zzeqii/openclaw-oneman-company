# 临时任务超时管理机制_v1.0
---
## 🔹 核心流程
### 1. 任务创建阶段
- 所有临时任务创建时必须同时配置**双倒计时告警**：
  - 交付前10分钟：进度提醒
  - 超时1分钟：自动触发告警流程
- 告警任务必须携带：`--to user:ou_108a6febb81880a62826958385ffe66c --channel feishu`参数，确保推送正确

### 2. 超时触发阶段
告警触发后自动执行以下动作：
1. **信息收集**：
   - 拉取任务当前进度、已完成内容、阻塞点
   - 整理需要决策的问题清单
2. **负责人询问**：
   - 向项目负责人（你）发送问询消息，格式：
     ```
     ⚠️ 任务超时问询：[任务名称]
     原定交付时间：[时间]
     当前进度：[X%]
     阻塞问题：[问题描述]
     需要决策：[待决策点清单]
     请确认：是否需要延期？新的交付时间？
     ```
3. **过程记录**：
   - 所有问询内容、回复内容、决策结果实时记录到`memory/[任务名称]_问询记录.md`
   - 记录包含：时间、问询内容、回复内容、决策结果、新交付时间

### 3. 结果反馈阶段
- 收到决策后1分钟内更新任务状态和告警时间
- 向你发送确认通知，格式：
  ```
  ✅ 任务延期已确认：[任务名称]
  新交付时间：[时间]
  决策内容：[摘要]
  已更新告警配置，到期将再次提醒
  ```

### 4. 任务完成阶段
- 任务交付并验收通过后，1分钟内自动删除对应的cron告警任务
- 问询记录归档到项目目录的`过程文档/`文件夹
