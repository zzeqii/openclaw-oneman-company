# 多模型调度配置方案_v1.0
---
## 🔹 配置结构
采用统一配置文件管理所有模型接入点，实现根据任务类型自动调度对应模型。

### 1. 配置文件（.env）
```env
# 通用配置
ARK_ENDPOINT=https://ark.cn-beijing.volces.com/api/v3

# 模型接入点配置
MODEL_CODE=ep-2024xxxxxxxxx-code  # 代码生成模型（ark-code-latest）
MODEL_IMAGE=ep-2024xxxxxxxxx-image # 图片生成模型（Seed Dream）
MODEL_VIDEO=ep-2024xxxxxxxxx-video # 视频生成模型（Seed Dance）
MODEL_CHAT=ep-2024xxxxxxxxx-chat   # 聊天/通用模型
```

### 2. 模型调度映射
| 任务类型 | 自动调度模型 | 能力范围 |
|---------|--------------|----------|
| 代码生成/调试/重构 | `MODEL_CODE` | 所有代码相关任务 |
| 图片生成/海报设计/素材制作 | `MODEL_IMAGE` | Seed Dream文生图/图生图 |
| 视频生成/数字人/动态素材 | `MODEL_VIDEO` | Seed Dance文生视频/图生视频 |
| 内容创作/文案/聊天/通用问答 | `MODEL_CHAT` | 所有文本类任务 |

## 🔹 调度逻辑实现
### 核心调度代码
```javascript
// model-scheduler.js
const modelConfig = {
  code: process.env.MODEL_CODE,
  image: process.env.MODEL_IMAGE,
  video: process.env.MODEL_VIDEO,
  chat: process.env.MODEL_CHAT,
};

/**
 * 根据任务类型获取对应模型接入点
 * @param {string} taskType 任务类型：code/image/video/chat
 * @returns {string} 模型接入点ID
 */
function getModelForTask(taskType) {
  const model = modelConfig[taskType];
  if (!model) {
    throw new Error(`不支持的任务类型：${taskType}，支持的类型：${Object.keys(modelConfig).join('/')}`);
  }
  return model;
}

module.exports = {
  getModelForTask,
};
```

### 调用示例
```javascript
// 生成图片时自动调用图片模型
const imageModel = getModelForTask('image');
const response = await axios.post(`${endpoint}/images/generations`, {
  model: imageModel,
  prompt: '一只可爱的熊猫',
  size: '512x512',
});

// 生成代码时自动调用代码模型
const codeModel = getModelForTask('code');
const response = await axios.post(`${endpoint}/chat/completions`, {
  model: codeModel,
  messages: [{ role: 'user', content: '写一个Python爬虫' }],
});

// 生成视频时自动调用视频模型
const videoModel = getModelForTask('video');
const response = await axios.post(`${endpoint}/video/generations`, {
  model: videoModel,
  prompt: '熊猫跳舞',
  duration: 5,
});
```

## 🔹 接入新模型步骤
1. 在方舟控制台接入新模型，获取接入点ID
2. 在`.env`文件中新增模型配置项，例如：
   ```env
   MODEL_AUDIO=ep-2024xxxxxxxxx-audio # 新增音频生成模型
   ```
3. 在`modelConfig`中新增映射：
   ```javascript
   const modelConfig = {
     // ... 现有模型
     audio: process.env.MODEL_AUDIO,
   };
   ```
4. 在调度映射表中新增任务类型即可。
---
## 🔹 优势
1. **统一管理**：所有模型配置集中在`.env`文件，修改方便
2. **自动调度**：根据任务类型自动匹配最优模型，无需手动指定
3. **易于扩展**：新增模型仅需3步，无需修改业务代码
4. **成本可控**：不同任务使用不同档位模型，避免高成本模型处理简单任务
