# Seed Dance视频生成功能问题排查报告
## 一、问题概述
Seed Dance视频生成功能调用时返回404错误，无法正常生成视频。Seed Dream图片生成功能正常。
## 二、排查过程
### 1. 功能测试
- ✅ Seed Dream图片生成：测试成功，可正常生成图片并保存到本地
- ❌ Seed Dance视频生成：调用时返回`Request failed with status code 404`错误
### 2. 根本原因分析
#### 问题1：API路径错误
当前代码使用的视频生成API路径为`/api/v3/video/generations`，根据火山引擎ARK最新文档，Seed Dance 1.5 Pro及以上版本的正确API路径应为：
- 提交任务：`POST /api/v3/videos/generations`（注意是videos复数，不是单数）
- 查询状态：`GET /api/v3/videos/generations/{task_id}`
#### 问题2：请求参数格式错误
Seed Dance新版本返回的字段名与旧版本不同：
- 旧版本：`taskStatus.video_url`
- 新版本：`taskStatus.data.video_url`
- 状态值：旧版本用`completed/failed`，新版本用`success/failed`
#### 问题3：SDK版本不匹配
当前使用的`@volcengine/ark` SDK版本与最新Seed Dance API不兼容，缺少官方SDK的`createVideoTask`、`getVideoTask`等封装方法。
#### 问题4：环境变量配置不完整
`.env`文件中缺少官方要求的`VOLC_ACCESS_KEY_ID`和`VOLC_ACCESS_KEY_SECRET`配置，仅配置了API密钥方式，而SDK需要AK/SK认证。
## 三、修复方案
### 1. 立即修复（API直接调用方式）
修改`ark-call.js`中的视频生成部分：
```javascript
// 提交视频生成任务
const submitResponse = await axios.post(`${endpoint}/videos/generations`, requestBody, {
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json',
  },
});
// 轮询任务状态
let taskStatus;
do {
  await new Promise(resolve => setTimeout(resolve, 10000)); // 每10秒轮询一次
  const statusResponse = await axios.get(`${endpoint}/videos/generations/${taskId}`, {
    headers: {
      'Authorization': `Bearer ${apiKey}`,
    },
  });
  taskStatus = statusResponse.data;
  console.log(`任务状态：${taskStatus.status}，进度：${taskStatus.progress || 0}%`);
} while (taskStatus.status !== 'success' && taskStatus.status !== 'failed');
if (taskStatus.status === 'failed') {
  throw new Error(`视频生成失败：${taskStatus.error || '未知错误'}`);
}
// 下载视频到本地
const videoUrl = taskStatus.video_url;
```
