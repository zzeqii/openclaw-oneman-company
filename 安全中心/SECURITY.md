# 安全中心
> 隐私合规与信息安全防护体系

## 🛡️ 核心安全规则（最高优先级，永久生效）

### 1. 隐私信息保护规则
- **绝对禁止**：代码中硬编码任何API Key、App Secret、Access Token、密码、私钥等敏感信息
- **存储要求**：所有敏感信息必须存储在 `.env*` 文件中，且 `.env* 必须加入 `.gitignore`
- **读取方式**：必须通过 `process.env` 读取，禁止提交到Git
- **泄漏处理**：发现泄漏立即执行「泄漏修复流程，必须清除Git历史，强制推送到远程覆盖

### 2. Git工作流安全规则
- **禁止直接推送到 main 分支**，所有变更必须走分支流程：
  1. 创建功能分支 `feature/xxx`
  2. 自动安全扫描确认无敏感信息
  3. 推送分支到远程
  4. 提交 Pull Request 合并到 main
  5. 安全中心Agent审核通过才能合并
- 每次推送分支前自动执行安全扫描：
  1. 扫描所有代码文件，检查是否有硬编码敏感信息
  2. 检查 `.env*` 是否在 `.gitignore` 中，确认没有已提交的env文件
  3. 发现问题禁止推送，立即启动修复
  4. 修复完成重新扫描，确认无问题才能推送

### 3. 敏感信息识别规则
以下关键词触发安全扫描：
- `APP_SECRET` / `app_secret` / `secret`
- `API_KEY` / `api_key` / `key`
- `ACCESS_TOKEN` / `access_token` / `token`
- `SECRET_KEY` / `private_key`
- `password` / `pwd`
- `AK` / `SK`

### 4. 泄漏修复流程
1. 立即从代码中移除硬编码，移到 `.env`
2. 更新 `.gitignore` 确保不被提交
3. 使用 `git filter-repo` 清除Git历史（如果已提交）
4. 强制推送覆盖远程历史
5. 记录泄漏事件到安全日志
6. 更新规则避免再犯

### 5. 定期扫描计划
- 每日提交自动扫描
- 每周日全量扫描整个项目库
- 发现问题立即汇报并修复
