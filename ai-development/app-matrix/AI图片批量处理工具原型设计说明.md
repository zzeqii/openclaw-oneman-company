# AI图片批量处理工具原型设计说明

## 1. 设计理念
- 极简主义：界面简洁，突出核心功能，减少干扰元素
- 效率优先：所有操作路径不超过3步，常用功能一键可达
- 可视化：处理流程、进度、效果对比直观可见
- 容错性：操作可撤销，错误可恢复，防止用户误操作

## 2. 整体架构
### 2.1 页面结构
```
┌─────────────────────────────────────────────────────────┐
│ 顶部导航栏：Logo、功能菜单、用户信息、通知中心          │
├───────────┬─────────────────────────────────────────────┤
│ 左侧侧边栏│ 主内容区域                                  │
│ - 首页    │                                             │
│ - 批量处理│ 动态加载对应功能页面                        │
│ - 模板市场│                                             │
│ - 任务中心│                                             │
│ - 历史记录│                                             │
│ - 系统设置│                                             │
└───────────┴─────────────────────────────────────────────┘
```

### 2.2 交互流程
1. 用户登录 → 首页概览
2. 选择处理功能 / 加载已有模板
3. 上传图片文件夹 / 选择云端存储路径
4. 配置处理参数 → 预览单张效果
5. 提交处理任务 → 实时查看进度
6. 处理完成 → 下载结果 / 导出报告

## 3. 核心页面设计
### 3.1 首页（Dashboard）
**布局：**
- 顶部：今日处理统计、任务完成率、节省时间统计卡片
- 中部：最近任务列表、常用模板快捷入口
- 底部：系统资源使用情况、待处理任务提醒

**核心元素：**
- 快速开始按钮：一键进入批量处理流程
- 模板推荐区：展示常用处理模板，点击即可使用
- 任务进度条：显示正在运行的任务实时进度

### 3.2 批量处理页面
**布局：三栏式设计**
```
┌────────────┬───────────────────┬────────────────────┐
│ 处理步骤   │ 参数配置区        │ 预览对比区         │
│ 1. 上传    │ 功能参数配置      │ 原图/处理后对比    │
│ 2. 配置    │ 流程编排          │ 放大查看细节       │
│ 3. 确认    │ 输出设置          │ 处理效果预览       │
└────────────┴───────────────────┴────────────────────┘
```

**交互设计：**
- 处理步骤：进度条显示当前步骤，支持点击回退
- 流程编排：拖拽式调整处理步骤顺序，可禁用/启用单个步骤
- 实时预览：修改参数后立即在预览区显示效果
- 批量预览：支持随机抽取多张图片预览处理效果

### 3.3 任务中心页面
**布局：**
- 顶部：任务筛选（全部/运行中/已完成/失败）、搜索框
- 中部：任务列表卡片，包含任务名称、状态、进度、开始时间、操作按钮
- 底部：分页控件、批量操作按钮

**核心功能：**
- 任务详情：点击任务卡片展开详细信息，包含处理步骤、参数、日志
- 任务操作：暂停/继续、取消、重试、复制任务、导出报告
- 批量操作：支持批量删除、批量导出、批量重试失败任务

### 3.4 模板管理页面
**布局：**
- 左侧：模板分类（基础处理、AI增强、自定义）
- 中部：模板卡片网格，展示模板名称、描述、使用次数、创建人
- 右侧：模板详情面板，包含参数说明、使用示例

**交互设计：**
- 一键使用：点击模板卡片直接进入批量处理页面，参数自动填充
- 模板编辑：支持修改模板参数，另存为新模板
- 模板分享：支持生成模板分享链接，团队成员可直接导入

## 4. 关键交互设计
### 4.1 拖拽上传
- 支持整个文件夹拖拽上传，自动识别子目录结构
- 上传过程中显示文件列表、进度、已上传大小
- 支持断点续传，大文件上传中断后可继续上传

### 4.2 对比预览
- 支持左右分栏对比、上下分栏对比、滑动对比三种模式
- 支持放大查看细节，最大放大比例400%
- 支持查看处理前后文件大小、分辨率、格式对比

### 4.3 进度提示
- 全局顶部进度条：显示当前任务总体进度
- 任务卡片进度环：直观显示单个任务完成百分比
- 处理完成提示：声音提醒+桌面通知，支持自定义通知方式

### 4.4 错误处理
- 处理失败的图片单独标记，显示错误原因和解决方案
- 支持单独重试失败的图片，无需重新处理整个批次
- 错误日志可导出，方便技术人员排查问题

## 5. 视觉设计规范
### 5.1 颜色体系
- 主色调：深蓝色 #165DFF（专业、科技感）
- 辅助色：
  - 成功：#00B42A
  - 警告：#FF7D00
  - 错误：#F53F3F
  - 信息：#86909C
- 中性色：
  - 背景：#F2F3F5
  - 卡片：#FFFFFF
  - 文本：#1D2129（主）、#4E5969（次）、#86909C（辅助）

### 5.2 字体规范
- 标题：Inter，16-24px，字重600
- 正文：Inter，14px，字重400
- 辅助文字：Inter，12px，字重400
- 按钮文字：Inter，14px，字重500

### 5.3 间距规范
- 页面边距：24px
- 卡片间距：16px
- 内容间距：12px
- 元素间距：8px

### 5.4 圆角规范
- 卡片圆角：8px
- 按钮圆角：6px
- 输入框圆角：6px
- 小元素圆角：4px

## 6. 响应式设计
### 6.1 桌面端（>=1200px）
- 完整三栏布局，所有功能完整展示
- 侧边栏固定宽度200px
- 内容区域自适应剩余宽度

### 6.2 平板端（768px-1199px）
- 侧边栏可折叠，折叠后宽度64px
- 两栏布局，参数配置区和预览区上下排列
- 保持所有功能可访问

### 6.3 移动端（<768px）
- 侧边栏转为底部导航
- 单栏布局，所有内容垂直排列
- 简化操作流程，突出核心功能
